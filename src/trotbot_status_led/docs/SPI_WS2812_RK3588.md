# RK3588：SPI（spidev）驱动 WS2812

## 原理简述

用 **SPI MOSI** 按固定时钟送出预先编码的字节流：每个 WS2812 数据位映射为 **1 个 SPI 字节**（8 个 SCLK 沿），使 MOSI 上出现近似 WS2812 NRZ 的高低占比。默认 **`spidev_max_speed_hz: 6400000`**（6.4 MHz）时，常见编码为：

- WS2812「0」→ **`0xC0`**
- WS2812「1」→ **`0xF8`**

帧尾再发送若干字节 **`0x00`**，拉长低电平，满足 WS2812 **RESET**（时长与字节数、SPI 时钟有关；可调 `spidev_reset_trailer_bytes`）。

若灯色整体偏移或发白，可在 YAML 里微调 **`spidev_max_speed_hz`** 以及 **`spidev_bit0_byte` / `spidev_bit1_byte`**（高级）。

## 鲁班猫 LubanCat-4-V1：Pin19 ↔ SPI0 MOSI（M2）

40PIN **Pin19** 复用 **`SPI0_MOSI_M2`**（表中 **`GPIO1_B2`**）。将灯带 **DIN** 接到 **Pin19**，**5V/GND** 仍按原配电（如 Pin2/Pin14）。**仅需连接 MOSI 到 DIN**；SPI 的 CLK/MISO/CS 若不接线可悬空（灯带不使用时钟线）。

启用 **`ws2812_backend: spidev`** 后，需在系统中出现 **`/dev/spidevX.Y`**（见下文设备树）。

## 设备树 / overlay：为何要开？

Linux 下 **`spidev`** 是挂在 **SPI 控制器**上的通用字符设备。若内核／镜像默认未启用对应 **SPI + spidev** 子节点，则 **`/dev/spidev*`** 不存在，节点会 WARN「打开失败」。

你需要做的通常是其一：

1. **使用板卡厂商提供的 SPI overlay**（鲁班猫 / RK3588 常见为 **`spi0`** 或编号的 **`fc600000.spi`** 等，以 **`dtb`/文档为准**），并启用 **`spidev`**；
2. **自建 overlay**：在目标 **`spi@…`** 节点下增加 **`spidev`** 兼容子节点，并把 **MOSI** 引脚配成 **`spi0_mosi`** 对应 **`m2`**（或厂商命名的 **`SPI0_MOSI_M2`**）分组。

具体寄存器基址、别名请以 **`/proc/device-tree`** 与 **`rk3588-lubancat-*.dts`** / 厂商 Wiki 为准；不同镜像 **`/dev/spidev`** 编号可能为 **`0.0`**、**`2.0`** 等，以实机为准：

```bash
ls -l /dev/spidev*
```

把 **`status_led_params.yaml`** 里的 **`spidev_path`** 改成实际路径。

### 权限与重启（`dialout` 仍打不开）

默认常见为 **`crw------- root root`**（**600**），仅 root 可访问；用户即使在 **`dialout`** 组也无效。临时 **`chmod 666`** 可验证，但**每次重启** udev 会恢复默认。

持久做法：安装仓库内 **[`config/udev/99-trotbot-spidev-dialout.rules`](../config/udev/99-trotbot-spidev-dialout.rules)**（`GROUP=dialout`, `MODE=0660`），执行 **`udevadm reload + trigger`**，并确保运行节点的用户在 **`dialout`** 组且**已重新登录**。

### 为何没有 `/dev/spidev*`，只有 `/dev/rkspi-dev*`？

鲁班猫提供的 **`rk3588-lubancat-spi0-m2-overlay.dtbo`** 里，子节点 **`compatible`** 为 **`rockchip,spidev`**（不是通用的 **`linux,spidev` / `spidev`**）。Rockchip **`spi-rockchip`** 驱动会为该节点注册 **misc 字符设备 `rkspi-devN`**，而 **不会** 创建传统的 **`/dev/spidevX.Y`**。

因此 **`ls /dev/spidev*`** 为空是**正常现象**。请执行：

```bash
ls -l /dev/rkspi*
```

- **`rkspi-dev2`**（若存在）在本机常见映射为 **`feb20000.spi`（SPI2）**，挂 **RK806 电源芯片**，**不是** 40PIN 上的 SPI0。
- 启用 **spi0-m2** 并重启后，通常会多出一个 **`rkspi-dev0`** 或 **`rkspi-dev1`**（编号依 probe 顺序而定），对应 **40PIN SPI0**。请在 **`/sys/bus/spi/devices/`** 下核对 **`spiX.Y`** 与 **`/proc/device-tree`** 中的 **`ffc60000.spi`**（SPI0 基址以板级 DTS 为准）。

本节点已对 **打开设备后 ioctl 失败** 的情况做兼容：**仍可用 `write()` 发送编码缓冲区**（与标准 spidev 相同）。若官方 overlay 里 **`spi-max-frequency`** 与默认编码假设的 **6.4 MHz** 不一致，灯色可能偏差，需微调 **`spidev_bit0_byte` / `spidev_bit1_byte`** 或改用自定义 overlay 固定时钟。

### 鲁班猫：ubuntuEnv.txt 追加官方 overlay（推荐）

在 **`/boot/firmware/ubuntuEnv.txt`** 的 **`overlays=`** 行末尾追加 **`rk3588-lubancat-spi0-m2-overlay`**（与其它 overlay 空格分隔），重启后应出现 **`/dev/spidev*`**。详见仓库 **`docs/trotbot/LubanCat-boot-spi0-overlay.md`**。

### overlay 示意（勿直接照搬：名称须与板级 DTS 一致）

以下仅为 **Rockchip 常见写法示意**，合并前请用 **`dtc -I dtb -O dts ...`** 对照你板子的 **`spi0`** 标签与 **pinctrl**：

```dts
/dts-v1/;
/plugin/;

/* 示意：在合适 fragment 中启用 spi0 + spidev；pinctrl 选 SPI0_M2 组到 Pin19 等 */
&spi0 {
    status = "okay";
    spidev@0 {
        compatible = "spidev";
        reg = <0>;
        spi-max-frequency = <10000000>;
    };
};
```

引脚复用 **`SPI0_MOSI_M2`** 需在 **`pinctrl`** 里选对 **`m2`** 分组（与 Pin19 一致）。若与 **UART4** 等冲突，需关掉占用同一组 pin 的外设或换 **`ws2812_backend`**。

## 权限

若 **`ls -l /dev/spidev0.0`** 显示属 **`root:root`** 且模式为 **`crw-------`**，普通用户 **open 会失败**，节点会 WARN「spidev 打开失败」。处理任选其一：

1. 临时：`sudo chmod 666 /dev/spidev0.0`（重启后可能还原）
2. 长期：用户加入 **`dialout`** 组；或在 **`/etc/udev/rules.d/`** 为 `spidev` 添加 **`MODE=0660`, `GROUP=dialout`** 规则后 **`udevadm trigger`**
3. **`sudo`** 运行 **`ros2 run`** / launch

一般将运行用户加入 **`dialout`** 组即可访问 **`/dev/spidev*`**（与 **`rockchip_mmap`** 不同）。

```bash
sudo usermod -aG dialout "$USER"
```

## ROS 参数（包内 `config/status_led_params.yaml`）

| 参数 | 含义 |
|------|------|
| `ws2812_backend` | 设为 **`spidev`** |
| `spidev_path` | 如 **`/dev/spidev0.0`** |
| `spidev_max_speed_hz` | 默认 **`6400000`** |
| `spidev_reset_trailer_bytes` | 默认 **`120`**，不够加长 reset 可调大 |
| `spidev_bit0_byte` / `spidev_bit1_byte` | 默认 **`-1`** 表示内置 **0xC0 / 0xF8** |
| `spidev_pre_write_idle_us` | 每次写 SPI 前空闲（µs），减轻首颗灯首比特误判（闪绿） |
| `spidev_leading_spi_zero_bytes` | 载荷前追加裸 **`0x00`** 字节数（实验性；不适则 **0**） |

首颗灯在加大 reset/pre、串电阻后仍偶发闪绿：多为 **物理层边际**，软件收益递减；跟踪见仓库 **`docs/trotbot/开发问题跟踪.md`** **ISSUE-WS2812-003**。
