# TrotBot 目录架构说明

这份文档用于帮助你快速理解 `src/trotbot` 包内的目录分层、关键文件作用，以及核心运行链路。

## 1. 包定位（一句话）

`trotbot` 是一个 **ROS 2 四足机器人应用包**：上层复用 CHAMP 做步态/状态估计，中层用 launch+config 组织系统，下层通过 `hal/` 把关节指令转为真实硬件（PCA9685 舵机）命令。

---

## 2. 顶层目录结构（按功能分组）

```text
src/trotbot/
├─ launch/                  # 启动编排：基础控制、遥控、舵机接口、CHAMP节点
├─ config/                  # 参数配置：CHAMP、手柄、仿真控制器
│  ├─ champ/
│  └─ ros_control/
├─ urdf/                    # 机器人模型（xacro/urdf）
├─ hal/                     # 硬件抽象层（Python）
│  ├─ interfaces/
│  ├─ hardware/
│  └─ utils/
├─ rviz/                    # RViz可视化配置
├─ scripts/                 # 调试/分析脚本
├─ test/                    # 测试脚本与代码风格检查
├─ patches/                 # 对外部依赖（champ_teleop）的补丁
├─ docs/                    # 文档资源（含图片说明）
│  └─ images/
├─ resource/                # ROS 2 包索引标记文件
├─ README.md                # 主说明文档（功能、部署、架构）
├─ 本地运行指南.md           # 更偏实操的本地部署与排障指南
├─ CMakeLists.txt           # ament_cmake 构建入口
├─ package.xml              # ROS 2 依赖声明
├─ setup.py / setup.cfg     # Python 打包与入口脚本
└─ LICENSE
```

## 3. 目录职责 + 关键文件说明

## `launch/`：系统启动编排层

- `trotbot_basic.launch.py`
  - 基础启动入口。
  - 负责拉起 CHAMP 控制与状态估计，并可选启动 `servo_interface`、`robot_state_publisher`、`rviz2`。
  - 常用开关：`use_servo_interface`、`rviz`、`description_file`、`gait_config_file`。
- `champ_controllers.launch.py`
  - 启动 CHAMP 的 `quadruped_controller_node` 与 `state_estimation_node`。
  - 将 `joints.yaml`、`links.yaml`、`gait.yaml` 注入 CHAMP。
- `servo_interface.launch.py`
  - 只启动硬件接口节点（`servo_interface`）。
- `trotbot_teleop.launch.py`
  - 启动 `champ_teleop`（键盘/手柄）。
  - 支持 `use_joystick`、`joystick_dev`、`use_xterm`。

## `config/`：参数与映射层

- `config/champ/joints.yaml`
  - CHAMP 关节名映射（四条腿各 3 个关键关节链路）。
- `config/champ/links.yaml`
  - CHAMP link 映射（base、大腿、小腿、足端）。
- `config/champ/gait.yaml`
  - 默认步态参数（速度上限、摆动高度、站姿高度等）。
- `config/champ/gait_minidog.yaml`
  - 面向 Minidog 几何的替代步态参数（尤其 nominal_height）。
- `config/champ_teleop_config.yaml`
  - `champ_teleop` 的速度、摇杆轴、按键映射。
- `config/gamepad_controller.yaml`
  - `joy_teleop` 风格手柄映射（慢速/快速模式）。
- `config/ros_control/trotbot_controller.yaml`
  - Gazebo/ros2_control 控制器参数模板（仿真用）。

## `urdf/`：模型与运动学几何基础

- `trotbot.urdf.xacro`
  - 主模型（链路、关节、惯性、Gazebo插件、ros2_control片段等）。
- `minidog_champ.urdf.xacro`
  - Minidog 几何版本，命名对齐 CHAMP。
  - 用于更换形态后快速复用控制栈。

## `hal/`：硬件抽象层（最关键）

### `hal/interfaces/`
- `servo_interface.py`
  - 订阅 `joint_group_effort_controller/joint_trajectory`。
  - 提取 CHAMP 输出的 12 关节角，做限位保护后交给硬件后端。

### `hal/hardware/`
- `hardware_interface.py`
  - 硬件后端总入口。
  - 优先使用 Adafruit PCA9685 后端，不可用时回退 sysfs PWM。
- `adafruit_servo_controller.py`
  - PCA9685 实际控制逻辑（I2C、通道、占空比换算）。
- `config.py`
  - PWM 通道映射、伺服参数、几何参数等。
- `servo_calibration.py`
  - 标定数据读写（`~/.config/trotbot/nvmem`）。
- `hardware_config.py`
  - 少量设备级配置（如手柄灯色等常量）。

### `hal/utils/`
- `calibrate_servos.py`
  - CLI 标定流程（逐关节校准中位和比例）。
- `calibrate_tool.py`
  - GUI 标定工具（滑条调节并写入标定）。
- `calibration_status.py`
  - 查看当前标定文件状态与参数。

## `scripts/`：调试与分析脚本

- `test_joystick.py`
  - 监听 `/joy`，验证手柄轴/按键映射是否匹配配置。
- `test_nominal_height.py`
  - 对不同站高做逆解可达性分析，辅助步态参数调优。

## `test/`：测试与验证

- `test_leg_comprehensive.py`
  - 大型交互测试脚本（单腿/多腿/IK/同步垂直动作）。
- `test_individual_joints.py`
  - 逐关节精细测试。
- `test_flake8.py`、`test_pep257.py`、`test_copyright.py`
  - 代码规范检查（ament 常规）。
- `README.md`
  - 测试方法与安全注意事项说明。

## `patches/`：上游依赖兼容补丁

- `apply_champ_teleop.sh`
  - 将本仓库维护的 `teleop.launch.py` 覆盖到 `champ_teleop`。
- `patches/champ_teleop/teleop.launch.py`
  - 修复/增强 `champ_teleop` 启动参数（如 `device_id`、`use_xterm`）。

## `rviz/` 与 `docs/`

- `rviz/trotbot.rviz`
  - 预设可视化布局，`RobotModel` 使用 `/robot_description` 话题。
- `docs/images/README.md`
  - 文档图片占位与命名说明（很多图是预留位）。

## 构建与包元信息

- `CMakeLists.txt`
  - 安装 launch/config/urdf/rviz 等资源，注册 Python 可执行入口。
- `package.xml`
  - 依赖声明（`rclpy`、`champ_base`、`champ_msgs`、`gazebo_ros` 等）。
- `setup.py`
  - Python 入口脚本定义：
    - `servo_interface`
    - `calibrate_servos`
    - `calibrate_tool`
    - `calibration_status`

## 4. 关键运行链路（你最常会遇到）

### 链路 A：基础运动控制（真机）

1. `trotbot_basic.launch.py` 启动 CHAMP 控制与状态估计。  
2. CHAMP 发布 `JointTrajectory` 到 `joint_group_effort_controller/joint_trajectory`。  
3. `hal/interfaces/servo_interface.py` 订阅该话题，做关节限位与矩阵重排。  
4. `hal/hardware/hardware_interface.py` 将角度转为 PWM 脉宽并下发 PCA9685。  
5. 舵机执行动作。  

### 链路 B：遥控输入到步态

1. `trotbot_teleop.launch.py` 启动 `champ_teleop`（键盘/手柄）。  
2. 遥控输入转为 `/cmd_vel`（及部分姿态控制）。  
3. CHAMP 将速度命令转换为步态关节轨迹，再进入链路 A。  

### 链路 C：可视化

1. `robot_state_publisher` 从 `urdf/*.xacro` 生成并发布 `/robot_description`。  
2. `rviz/trotbot.rviz` 预设 RobotModel + TF 显示。

## 5. 你后续最常改的“入口文件”

- **换机器人几何/尺寸**：`urdf/` + `config/champ/{joints,links,gait}*.yaml`
- **换底层执行器（非舵机）**：`hal/hardware/hardware_interface.py` 及后端文件
- **调步态手感**：`config/champ/gait*.yaml`
- **调手柄映射**：`config/gamepad_controller.yaml` 与 `config/champ_teleop_config.yaml`
- **调启动行为**：`launch/trotbot_basic.launch.py`、`launch/trotbot_teleop.launch.py`

---

## 6. 与工作空间其它仓库的关系

`trotbot` 不是独立闭环，至少依赖：

- `champ`（提供 `champ_base`、`champ_msgs`）
- `champ_teleop`（遥控节点）

也就是：`trotbot` 主要做“机器人定制层”和“硬件落地层”，核心步态控制本体来自 CHAMP。

---

## 7. 快速阅读建议（10 分钟上手版）

如果你想最短时间建立全局认知，建议按这个顺序读：

1. `launch/trotbot_basic.launch.py`
2. `launch/champ_controllers.launch.py`
3. `hal/interfaces/servo_interface.py`
4. `hal/hardware/hardware_interface.py`
5. `config/champ/joints.yaml` + `links.yaml` + `gait.yaml`
6. `urdf/trotbot.urdf.xacro`（或 `minidog_champ.urdf.xacro`）

这样可以先把“控制链路”读通，再按需下钻具体模块。
