# TrotBot 双 CAN 桥接开发计划（LubanCat-4-V1）

本文档用于管理研发过程中的阶段任务、验收标准与风险项。  
稳定执行步骤请查看：`本地运行指南.md`。

---

## 1. 项目目标

- 平台：`LubanCat-4-V1`（外接两路 CAN 收发器）
- 总线：`can0` 控前腿、`can1` 控后腿
- 波特率：`1 Mbps`
- 目标：将 CHAMP 输出的关节位置转换为电机 CAN 指令，完成四足基础步态闭环运行

---

## 2. 架构与边界

- 上层保持不变：`champ_base`、`quadruped_controller_node`
- 新增中间层：`trotbot_can_bridge`（订阅 `/joint_group_effort_controller/joint_trajectory`）
- 下层执行：SocketCAN -> 灵足电机
- 反馈回传：电机反馈 -> `/joint_states`（或桥接内状态）
- 现有 `servo_interface` 在真机 CAN 模式下关闭

---

## 3. 分阶段计划

### P0：硬件与链路打通（最小风险）

- [ ] 外接两路 CAN 收发器，确认供电、共地、终端电阻
- [ ] 开启板卡 CAN overlay，重启后识别 `can0/can1`
- [ ] 配置 1M 波特率并验证：
  - [ ] `ip -details link show can0`
  - [ ] `ip -details link show can1`
- [ ] 使用 `cansend`/`candump` 完成收发验证（单路、双路）

**验收标准**
- 两路 CAN 都能稳定收发，连续 10 分钟无明显掉线或总线错误

### P1：最小桥接（先跑起来）

- [ ] 创建 ROS2 包 `trotbot_can_bridge`
- [ ] 订阅 `/joint_group_effort_controller/joint_trajectory`
- [ ] 实现最小五元组：
  - [ ] `p = positions`
  - [ ] `kp/kd = 固定常量`
  - [ ] `v=0`
  - [ ] `tau=0`
- [ ] 按 `joint -> (canX, motor_id)` 映射下发
- [ ] `candump can0/can1` 核对帧 ID、长度、频率

**验收标准**
- 指定关节在低速小幅命令下可重复运动，无明显误触发

### P2：零位与方向标定（可控可复现）

- [ ] 确定统一标定姿态（中立站姿/吊装中立）
- [ ] 建立每关节参数表：
  - [ ] `sign`
  - [ ] `joint_zero_offset_rad`
  - [ ] 安全角限幅
- [ ] 应用转换公式：
  - [ ] `motor_pos_rad = sign * urdf_pos_rad + joint_zero_offset_rad`
- [ ] 增加越限保护、掉线保护

**验收标准**
- 12 关节在中立姿态误差可控，动作方向全部正确，无硬限位碰撞

### P3：反馈闭环与稳定性

- [ ] 解析电机反馈帧（位置/速度/力矩/状态字）
- [ ] 发布 `/joint_states`（或等价状态话题）
- [ ] 补充急停、上电顺序、故障恢复策略
- [ ] 长时间稳定性测试（连续步态运行）

**验收标准**
- 可稳定运行基础步态；异常时能安全降级或停机

---

## 4. 关键配置文件建议

建议在 `trotbot_can_bridge/config/` 下维护：

- `motor_map.yaml`：关节到 CAN 接口与 ID
- `calibration.yaml`：`sign`、`joint_zero_offset_rad`、关节限幅
- `control_gains.yaml`：`kp/kd`（可先全局，后续细化到关节）

- [ ] 12 个关节名与 CHAMP `joint_names` 对应表
- [ ] 每关节 CAN ID 与总线归属（can0/can1）

---

## 6. 风险与缓解

- 协议“相似”不等于 bit 级兼容  
  -> 必须用厂家协议文档 + `candump` 抓包逐字段核对

- Linux 非实时抖动影响高频闭环  
  -> 前期先降频求稳；后续再评估实时线程或下沉 MCU

- 零位不一致导致动作方向错或撞限位  
  -> 先做单关节低幅试转，再开放多关节同步

---

## 7. 里程碑判断（是否接近“可正常运动”）

满足以下条件后，基本可进入“正常运动”阶段：

- [ ] P0/P1/P2 全部通过
- [ ] 全关节方向与零位一致
- [ ] 步态切换无明显失稳
- [ ] 故障时可安全停机

说明：这时可认为“功能跑通”；若要提高可靠性，还需继续完成 P3 的稳定性与保护策略。
