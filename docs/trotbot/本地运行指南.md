## 实机 EL05 双 CAN + 遥控（当前推荐基线）

1. **编译与环境**（至少包含 `trotbot`、`trotbot_can_bridge`、`champ_base`、`champ_teleop`）：
   ```bash
   source /opt/ros/humble/setup.bash
   cd ~/trotbot_ws
   colcon build --symlink-install --packages-select champ_msgs champ champ_base champ_teleop trotbot trotbot_can_bridge
   source install/setup.bash
   ```
2. **CAN**：确认 `can0`/`can1` 已 `up`、波特率与电机一致（常见 1Mbps）。参见根目录 **`README.md`**「MIT 双 CAN」章节与 `src/trotbot/本地运行指南.md` 11.7 节。
3. **整机启动**（步态 + CAN 桥 + 上电状态机，默认 **无** `servo_interface` 时勿再依赖舵机）：
   ```bash
   ros2 launch trotbot trotbot_basic.launch.py use_servo_interface:=false use_can_bridge:=true
   ```
4. **遥控**（另一终端，`champ_teleop` + 手柄/键盘）：
   ```bash
   ros2 launch trotbot trotbot_teleop.launch.py use_joystick:=true use_xterm:=false
   ```
5. **启停与姿态**：由 **`power_sequence_node`** 处理（默认随 `can_bridge.launch.py` 启动）。手柄约定见 **`src/trotbot_can_bridge/config/power_sequence.yaml`** 与根 **`README.md` 2a-1**；无手柄时可用 `/power_sequence/command`（`start` / `prone` / `shutdown`）。
6. **RViz 双模型**：`trotbot_basic.launch.py` 设 `rviz:=true` 时加载 `share/trotbot/rviz/trotbot.rviz`；反馈模型订阅 `/robot_description_feedback` 与 `fb/` TF 前缀，依赖 **`/joint_states_feedback`**（`motor_protocol_node` 定时发布 + CAN 更新缓存）。
