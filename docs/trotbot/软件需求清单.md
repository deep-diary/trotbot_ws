# TrotBot 软件需求清单（按优先级）

> 本文档仅保留软件需求与实现状态，不含排期/人天/里程碑计划。  
> 状态定义：`[x] 已完成`（代码已实现），`[ ] 未完成`（当前仓库未实现或未闭环）。

---

## 1. 范围与目标

- 范围：`CHAMP -> CAN Bridge -> 电机反馈 -> RViz/状态话题`。
- 目标：形成可逐条开发与验收的软件需求清单，优先保证安全与可控，再提升可观测与易用性。

---

## 2. 高优先级（P0，先做）

- [x] **RQ-001 关节命名映射与顺序一致性**  
  - 说明：桥接层按 CHAMP 关节名顺序映射 12 关节。  
  - 代码依据：`trotbot_can_bridge/include/trotbot_can_bridge/dog_mapper.hpp` 中 `kChampJointNames`。

- [x] **RQ-002 关节到 CAN 总线与电机 ID 映射**  
  - 说明：12 关节固定映射到 `11/12/13/21/22/23/51/52/53/61/62/63`，前腿 `can0`，后腿 `can1`。  
  - 代码依据：`dog_mapper.hpp`、`trotbot_can_bridge/config/motor_map.yaml`。

- [x] **RQ-003 URDF角 -> 电机角 正向转换**  
  - 说明：按 `motor_pos_rad = sign * urdf_pos_rad + offset` 转换后下发。  
  - 代码依据：`trotbot_can_bridge/src/motor_protocol_node.cpp` 的 `mapped_position` 计算。

- [x] **RQ-004 标定参数入口（sign/offset）**  
  - 说明：支持 `joint_signs[12]`、`joint_offsets_rad[12]` 参数化。  
  - 代码依据：`motor_protocol_node.cpp` 参数声明；`trotbot_can_bridge/config/control_gains.yaml`。

- [x] **RQ-005 电机角 -> URDF角 反向转换并发布标准状态**  
  - 说明：把反馈机械角反解为 URDF 角并发布标准 `JointState`。  
  - 当前实现：发布到 `/joint_states_feedback`（等价标准状态话题），用于实时反馈模型。  
  - 代码依据：`motor_protocol_node.cpp` 中 `champ_feedback = (current_angle - offset) / sign` 与 `PublishFeedbackJointStates()`；`control_gains.yaml` 中 `publish_feedback_joint_states`、`feedback_joint_states_topic`。

- [x] **RQ-006 机械限位保护（按 URDF 限位映射到电机限位）**  
  - 说明：将 URDF 限位映射到电机域并执行发送前截断保护。  
  - 代码依据：`motor_protocol_node.cpp` 的 `ClampJointCommand()`、`RebuildMotorLimitTable()`、`ClampMotorCommand()`；`control_gains.yaml` 的 `joint_cmd_min_rad/max_rad`。

- [x] **RQ-007 上电初始化流程编排（安全启动）**  
  - 说明：`（可选）set_zero -> 前置校验 -> init(使能+MIT首帧0, 主动上报默认关闭) -> 启动门禁短窗 -> 软启动站立`。  
  - 当前状态：已实现 `power_sequence_node` 状态机与 `motor_protocol_node` 启动门禁联动；默认由遥控组合键触发。  
  - 启动编排：`trotbot_basic.launch.py` / `champ_controllers.launch.py` 默认 **`description_file=minidog_champ.urdf.xacro`**、**`gait_config_file=gait_minidog.yaml`**，避免误用旧款 `trotbot.urdf.xacro` 与实机 MIT 腿长不一致；旧整机显式传参覆盖。

- [x] **RQ-008 下电收拢流程编排（安全停止）**  
  - 说明：平滑趴下（z 轴分段下降） -> 到位判定（高度/时间窗） -> 12 电机失能 -> 退出控制门禁；**趴下保持使能**与**下电**拆分：`prone`/Circle → `ProneHold`；`shutdown`/L1+R1+Share → `SoftProne`→`Disable`。  
  - 当前状态：已实现并接入遥控与 `/power_sequence/command`。

## 3. 中优先级（P1，P0稳定后）

- [x] **RQ-009 MIT 控制参数化下发**  
  - 说明：支持 `kp/kd/default_velocity/default_tau_ff` 参数。  
  - 代码依据：`motor_protocol_node.cpp` 参数声明与 `BuildMitControlFrame` 调用。

- [x] **RQ-010 镜像参数展开能力**  
  - 说明：支持由 `hip/thigh/calf` 三参数展开 12 维偏移。  
  - 代码依据：`BuildMirrorJointOffsetsRad` 与 `expand_joint_offsets_from_mirror`。

- [x] **RQ-011 RViz 双模型（指令态 + 实时态）**  
  - 说明：同屏显示“指令狗”与“实时狗”。  
  - 代码依据：`trotbot/rviz/trotbot.rviz` 中 `RobotModel` + `RobotModelFeedback`；`trotbot_basic.launch.py` 中 `robot_state_publisher_feedback` 与 `/joint_states_feedback` remap。

- [ ] **RQ-012 反馈结构化发布（位置/速度/力矩/状态字）**  
  - 说明：需要从字符串日志升级为结构化话题，便于算法与监控复用。  
  - 当前状态：已具备 `/joint_states_feedback`（结构化位置），但速度/力矩/状态字仍主要在 `/motor_feedback` 字符串中，未形成统一结构化消息。

- [x] **RQ-016 运行时 MIT 增益在线调参**  
  - 说明：支持运行时按全局/前后腿动态调整 `kp/kd/tau`。  
  - 代码依据：`motor_protocol_node.cpp` 的 `runtime_tune_topic`、`OnTuneCommand()` 与 `runtime_kp/kd/tau_can0/can1`。

- [x] **RQ-017 启动与模式切换平滑控制**  
  - 说明：支持上电后平滑与电机模式上升沿平滑锚点重置，降低突跳风险。  
  - 代码依据：`enable_startup_smoothing`、`startup_smoothing_duration_s`、`enable_mode_rising_smoothing` 与 `ResetSmoothingForJoint()`。

- [x] **RQ-018 基于反馈的新鲜度与步进限幅保护**  
  - 说明：当反馈新鲜且变化过大时限制单周期目标步进，降低瞬态冲击。  
  - 代码依据：`enable_feedback_step_limit`、`feedback_step_limit_rad`、`feedback_fresh_timeout_s` 与 `SmoothJointCommand()` 分支逻辑。
