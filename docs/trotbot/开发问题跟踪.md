# TrotBot 开发问题跟踪

本文档用于记录开发、联调、实测过程中遇到的问题，并持续跟踪定位、修复、回归结果。

适用场景：
- 现场实测异常（动作异常、总线异常、姿态异常等）
- 开发过程问题（参数不生效、接口不一致、启动失败等）
- 回归阶段问题（修复后复现、边界条件失败、偶发故障）

建议每条问题至少包含以下信息：
- 问题ID
- 发现时间
- 发现版本/分支
- 现象描述
- 复现步骤
- 期望结果
- 实际结果
- 影响范围
- 临时绕过方案
- 根因分析
- 修复方案
- 回归结论
- 当前状态（Open / In Progress / Resolved / Closed）
